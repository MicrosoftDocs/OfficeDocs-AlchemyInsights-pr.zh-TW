# <a name="error-404-when-accessing-files-in-sharepoint-or-onedrive"></a>存取 SharePoint 或 OneDrive 中的檔案時, 發生錯誤404

當使用者嘗試存取 SharePoint 或 OneDrive 中的網站或檔案時, 會收到**錯誤 404** 。 這通常是因為網站或檔案或群組被重新命名、移動或刪除。
使用者會在嘗試存取根網站集合時遇到404錯誤, 且已被刪除。

若要解決已重新命名、移動或刪除之網站的錯誤 404, 請參閱[還原已刪除的網站集合](https://docs.microsoft.com/sharepoint/restore-deleted-site-collection)。

對於位於新系統管理中心預覽中的新式網站 (通訊、群組連線或其他網站), 請參閱在[新的 SharePoint 系統管理中心中查看及還原刪除的網站](https://docs.microsoft.com/sharepoint/view-and-restore-deleted-sites-in-new-admin-center)。

若要解析已重新命名、移動或刪除的檔案 (或其他專案), 請移至 SharePoint 或 OneDrive 網站, 並從網站內容中查看回收站, 請參閱[還原 SharePoint 網站回收站中的專案](https://support.office.com/article/Restore-items-in-the-Recycle-Bin-of-a-SharePoint-site-6df466b6-55f2-4898-8d6e-c0dff851a0be)。

 如果您仍無法找到該專案, 您可以在啟用記錄的情況下搜尋 audit 記錄檔, 請參閱[search the audit log in The Office &amp; 365 Security 合規性中心](https://support.office.com/client/search-the-audit-log-in-the-office-365-security-compliance-center-0d4d0f35-390b-4518-800e-0c7ec95e946c)。
